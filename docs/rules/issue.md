# Issueの書き方

---

## タイトル
```
[type] [domain] 概要
```
例：
- [Feature] [API] ログイン機能追加
- [Fix] [WEB] クラッシュ修正

---

## type

- Feature 機能追加
- Enhancement 機能改良・拡張
- Fix バグ修正
- Refactor 内部改善
- Chore その他

---

## domain

- API バックエンド
- WEB フロントエンド
- DB データベース
- INFRA インフラ
- COMMON 共通

---

## ルール
- 1 Issue = 1目的

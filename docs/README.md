# ドキュメント一覧

このディレクトリは、開発に必要なルール・設計・意思決定をまとめたものです。  
初めて参加する場合は「onboarding」から確認してください。

---

## onboarding/
開発に参加するための初期ガイド

- 環境構築手順
- 開発フロー（Issue → PR）
- よくあるトラブルと対処

---

## rules/
開発ルール

- Issueの書き方
- ブランチ・コミット規則
- PRルール
- type / domain 定義

---

## architecture/
システム全体の設計

- 構成概要
- 技術構成
- データの流れ

---

## adr/
設計上の意思決定の記録（Architecture Decision Record）

- なぜその設計を選んだか
- どのような選択肢があったか
- 影響範囲

---

## features/
機能ごとの設計

- 各機能の仕様
- 挙動・ロジック
- エッジケース

---

## api/
API仕様

- エンドポイント一覧
- リクエスト / レスポンス
- エラー仕様

---

## glossary/
用語集

- プロジェクト内で使用する用語の定義
